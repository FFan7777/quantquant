## 一、现状诊断

### 1.1 性能瓶颈分析

| 层次 | 当前状态 | 瓶颈？ |
|------|---------|--------|
| **择时模型** | MA60状态预测，槽分布: 0=78次/10=12次/20=52次 | ❌ 已近最优（3年实测近乎完美的多空分离） |

## 二、优化方向全景（无数据泄漏）

### 方向A：Walk-Forward Training（滚动训练）⭐⭐⭐ 优先级最高

**原理**：当前使用单次训练（2018-2022），3年测试期始终用同一个模型。WFO允许模型每N个月用最新数据重训，适应市场regime变化。

**数据泄漏风险**：**无** — 在时间T进行预测时，仅使用T-embargo_days之前的数据训练。

**实现方案**：
```
扩展窗口WFO（每半年扩展一次）：
  - 2023-02 预测：训练 2018-01 ~ 2022-12（当前已有）
  - 2023-08 预测：训练 2018-01 ~ 2023-06（+6个月数据）
  - 2024-02 预测：训练 2018-01 ~ 2023-12（+12个月数据）
  - 2024-08 预测：训练 2018-01 ~ 2024-06
  - 2025-02 预测：训练 2018-01 ~ 2024-12
```

**预期收益**：2023年的市场模式可以在2024年被学习，2024年的模式可以在2025年被学习。特别是对2023低仓期后的拐点识别有帮助。

**实现复杂度**：中（需要修改 xgboost_cross_section.py 支持 WFO 模式，输出每日预测而非一次性训练）

---

### 方向B：动态仓位大小（基于信号强度）⭐⭐⭐ 快速见效

**原理**：当前每个槽位大小固定 = 总净值/MAX_SLOTS（不管预测分数高低）。改为按信号强度分配，排名第1的股票得到更多资金。

**数据泄漏风险**：**无** — 仅使用当期预测分数，不引入未来数据。

**实现方案（凸权重）**：
```python
# 选出 top_n 只股票后，用 rank 分数做凸权重
# pred_rank 是截面内百分位 [0,1]
# 权重 = softmax(pred_rank * temperature) 归一化
# temperature=2 时：第1名约是第8名的2x仓位
```

**约束**：
- 单股权重上限 = 3/MAX_SLOTS（防止过度集中）
- 单股权重下限 = 0.3/MAX_SLOTS（保证每股有最低参与量）

**预期收益**：让强信号股票有更大敞口，弱信号股票减少持仓，理论上提升IC → 收益转化率。

**实现复杂度**：低（仅修改 index_ma_combined_strategy.py 中的 rebalance 函数）

---

### 方向C：置信度门槛过滤（仅持高排名股票）⭐⭐

**原理**：当前选 top-MAX_SLOTS 只股票，不管信号强弱。改为只持信号排名 > 某阈值（如top 5%截面）的股票。

**现象分析**：
- 若仓位利用率只有44%，说明安全过滤器已经是瓶颈
- 但如果把阈值从top-8改为top-5%，只持"确信股"
- 期望：减少持仓数，但每只股票alpha更高

**数据泄漏风险**：**无**

**实现复杂度**：低（在 load_cs_predictions / rebalance 中加预测分数阈值）

---

### 方向D：LightGBM Ensemble ⭐⭐

**原理**：XGBoost 和 LightGBM 在相同特征上倾向于捕捉不同非线性模式。Ensemble 两个算法的预测可以降低方差。

**数据泄漏风险**：**无** — 同样使用训练/测试切分，同样遵守embargo。

**实现方案**：
```python
# 在 train_model() 中同时训练 xgb + lgbm
# pred = 0.5 * rank(xgb_pred) + 0.5 * rank(lgbm_pred)
# 可以进一步与H5 ensemble: pred = 0.5*rank(xgb_h10) + 0.25*rank(lgbm_h10) + 0.25*rank(h5)
```

**实现复杂度**：低（新增 lightgbm 训练，加权融合）

---

### 方向E：风险调整标签 ⭐⭐

**原理**：当前标签 = 个股10日收益 - 基准10日收益（截面百分位）。改为风险调整版本：标签 = 超额收益 / 同期已实现波动率（Sharpe-per-stock）。

**动机**：模型可能学会选高beta股票（在牛市好用，在熊市亏损）。风险调整标签让模型学习高质量Alpha，而非高风险溢价。

**数据泄漏风险**：
- ⚠️ **需要谨慎**：标签中的波动率如果用未来20日波动率 → 泄漏！
- ✅ **安全做法**：用前期（历史）波动率，例如 `vol_20d`（已有特征，当日已知）
- 公式：`label_adj = excess_ret_10d / vol_20d`（分母是已知量，无泄漏）

**实现复杂度**：低（修改标签计算部分）

---

### 方向F：多标签融合（分类+回归）⭐

**原理**：同时训练一个回归模型（预测超额收益大小）和一个分类模型（预测超额收益是否>0），用分类概率过滤回归输出。

**数据泄漏风险**：**无** — 同样使用历史数据训练。

**实现复杂度**：中（需要额外训练分类模型）

---

### 方向G：行业动量特征（截面内）⭐

**原理**：在截面内，加入"行业近N日超额收益相对大盘"特征，让模型感知行业轮动。

**数据泄漏风险**：
- ✅ **安全**：行业过去N日收益是已知量
- 但需确保行业分类数据（stock_basic.industry）不使用未来调整后的行业分类

**实现复杂度**：中（需要计算每只股票所在行业的历史平均收益）

---

### 方向H：已放弃方向（禁止重复尝试）

| 方向 | 原因 |
|------|------|
| MA20/MA60交叉择时 | 已测试，年化↓（错过2023-2025拐点） |
| 动态止损（熊市收紧） | 已测试，whipsaw↑ |
| F-Score ≥ 7 | 已测试，集中度风险↑ |
| 入场加close>MA60过滤 | 已测试，熊市跌幅更大 |
| H20模型 | 已测试，时域不匹配（5日调仓+20日标签） |
| SLOT_CONFIRM 3→1或2 | 已测试，whipsaw↑ |
| 行业动量特征（中性化后） | 已测试，信息丢失 |
| 动量特征(ret_250d) | 已测试，2024-2025价值反转市场不适用 |

## 三、实施计划

### 优先级排序（风险回报比）

| 优先级 | 方向 | 预期Sharpe提升 | 实现成本 | 状态 |
|--------|------|--------------|---------|------|
| 🥇 1 | **B: 动态仓位大小** | +0.1~0.2 | 低 | 待实现 |
| 🥈 2 | **A: Walk-Forward Training** | +0.2~0.4 | 中 | 待实现 |
| 🥉 3 | **D: LightGBM Ensemble** | +0.05~0.15 | 低 | 待实现 |
| 4 | **E: 风险调整标签** | +0.05~0.1 | 低 | 待实现 |
| 5 | **C: 置信度门槛** | ±0 | 低 | 待实现（需观察仓位利用率） |

### 实施顺序

1. **先实现B（动态仓位）** — 不需要重新训练模型，改动小，风险低
2. **再实现D（LightGBM）** — 新增训练约30分钟，快速验证ensemble效果
3. **最后实现A（WFO）** — 最复杂，但潜力最大；需要5次重训，约20分钟×5=100分钟

## 四、数据泄漏检查清单

每次修改后必须确认：

- [ ] 所有财务数据使用 `f_ann_date`（第一披露日），非 `ann_date` 或 `end_date`
- [ ] 标签使用未来数据，但**仅在标签计算时**（不混入特征）
- [ ] 训练集截止 ≤ 测试集起点 - embargo_days（当前20个交易日）
- [ ] WFO中，预测日期T的模型训练数据截止 ≤ T - 20个交易日
- [ ] 没有使用截面排名的未来信息（如"这是本月最好的股票"）
- [ ] 风险调整标签分母（vol_20d）是已知历史波动率，非未来波动率
