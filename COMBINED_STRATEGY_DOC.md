# 指数择时 + 截面选股 联合量化策略 技术文档

**文件**: `index_ma_combined_strategy.py`
**运行**: `python index_ma_combined_strategy.py`（约 1-2 分钟）

---

## 一、核心思路

本策略将**大盘择时**与**个股选股**解耦，分别用两个独立模型完成，最后在仓位层面合并：

```
大盘择时（index_timing_model.py）
    ↓ slots ∈ {0, 10, 20}  →  决定"是否开仓"及"最大持仓数"
截面选股（xgboost_cross_section.py）
    ↓ pred ∈ [0,1]         →  决定"开哪几只"
风险平价（20日波动率反比加权）
    ↓                      →  决定"每只开多少"
```

**直觉**：大盘不好时宁可空仓（槽位归零），大盘好时让 CS 模型找最强个股，同时用风险平价控制各头寸的风险贡献均等。

## 二、模块一：指数择时

### 信号来源

`output/csv/index_timing_predictions.csv`（由 `index_timing_model.py --label_type ma60_state` 生成）

### 信号含义

| slots 值 | 含义 | 操作 |
|----------|------|------|
| 0 | 熊市 / 空仓 | 不新开仓，现有持仓由风控自然退出 |
| 10 | 半仓（CSI300 在 MA20~MA60 之间） | 最多持有 `HALF_SLOTS=3` 只 |
| 20 | 满仓（CSI300 在 MA60 之上） | 最多持有 `MAX_SLOTS=8` 只 |

### 硬覆盖规则（已内置于择时模型输出中）

- `CSI300 < MA20` → 强制 slots=0，停止一切新开仓
- `MA20 ≤ CSI300 < MA60` → slots=10（半仓）
- `CSI300 ≥ MA60 且 ML 看涨` → slots=20（满仓）

### 择时准确性（2023-2025 样本外验证）

| 年份 | slots=0 日均 CSI300 涨跌 | slots>0 日均 CSI300 涨跌 |
|------|--------------------------|--------------------------|
| 2023 | -0.238%（累计 -31%） | +0.232~+0.336%（累计 +8~19%） |
| 2024 | -0.136%（累计 -19%） | +0.121~+0.482%（累计 +2~39%） |
| 2025 | -0.193%（累计 -18%） | +0.247~+0.372%（累计 +4~38%） |

> 结论：择时模型已达到近似完美的市场状态分离，是本策略的核心护盾。

## 三、模块二：截面选股

### 信号来源

`output/csv/xgb_cross_section_predictions.csv`（H10 主模型）
`output/csv/xgb_cs_pred_h5.csv`（H5 辅助模型，用于 Ensemble）

### Ensemble 融合方式

```python
r10 = cross_sectional_rank(pred_h10)   # H10 截面排名百分位
r5  = cross_sectional_rank(pred_h5)    # H5 截面排名百分位
pred_final = 0.7 × r10 + 0.3 × r5    # 加权融合
```

- **为何 Ensemble 有效**：H10 预测 10 日窗口，H5 捕捉更短期反转信号，两者互补
- **为何 70/30 而非 50/50**：H10 与调仓频率（5日）更一致；H5 作为短期修正权重不宜过大
- **稳定性**：XGBoost 因 `n_jobs=-1` 存在非确定性，单次 ICIR 在 0.983~1.009 之间波动；Ensemble 可将年化收益的标准差从 ±5% 压缩至 ±2%

### CS 模型关键指标（测试集 2023-2025）

| 指标 | H10 | H5 |
|------|-----|-----|
| Rank IC 均值 | +0.113 | +0.105 |
| IC 正比率 | 83.6% | ~80% |
| ICIR | ~1.00 | ~0.94 |
| GAUC | 0.552 | ~0.548 |

## 四、仓位管理

### 槽位权重机制

每个持仓的**目标价值 = 总净值 / MAX_SLOTS**，固定不随持仓数量变化：

```
slot_value_base = total_portfolio_value / MAX_SLOTS
```

这样做的好处：新增/退出一只股票不会触发其他持仓的级联再平衡，大幅减少交易摩擦。

### 风险平价加权（新增）

开新仓时，在基础槽位上叠加波动率反比缩放：

```python
scale = clip(median_vol_20d / this_stock_vol_20d, 0.5, 2.0)
slot_value = slot_value_base × scale
```

- 低波动率股票：获得更多资金（最多 2×）
- 高波动率股票：获得更少资金（最少 0.5×）
- 预期各头寸风险贡献趋于均等，提升 Sharpe 比率

### 闲置资金收益

```python
pf.cash *= (1 + RISK_FREE_RATE / 252)   # RISK_FREE_RATE = 0.02
```

空仓/半仓期间现金按 **2% 年化**增值（模拟货币基金或短债）。
slots=0 约占全部交易日的 40%~50%，此项贡献约 **+0.8~1.2% 年化**。

### 实际最大持仓数映射

```python
actual_slots = MAX_SLOTS if slots_today == 20 else HALF_SLOTS
# slots=20 → 最多 8 只
# slots=10 → 最多 3 只（真正的半仓）
```

## 五、个股风险管理

每个交易日收盘检查所有持仓（`daily_stop_check`）：

| 触发条件 | 阈值 | 说明 |
|----------|------|------|
| 硬止损（入场价回撤） | 8% | 立即清仓，不受最短持有期限制 |
| 追踪止损（峰值回撤） | 100%（禁用） | 依赖 MA 死叉自然退出 |
| MA 死叉 | MA5 < MA20 连续 **5** 天 | 需持仓满 **5** 个交易日后才检查 |
| 调仓时淘汰 | 不在 top-N 且持满 MIN_HOLD_DAYS | 在调仓日被新股替换 |

### 防 Whipsaw 机制

- `SLOT_CONFIRM_DAYS = 3`：slots 连续 3 天 > 0 才允许新开仓，防止信号切换时频繁进出
- `MIN_HOLD_DAYS = 5`：MA 死叉检查需持仓满 5 日，避免短暂噪声触发止损
- `MA_DEATH_DAYS = 5`：MA5 < MA20 需连续 5 天才确认死叉

| 方案 | 结果 | 原因 |

## 十、架构要点

### 为何不用 MA 直接择时？

纯 MA 策略（如 MA20/MA60 均线交叉）存在滞后性：市场已下跌 10% 才确认死叉。本策略用 **ML 预测"15日后是否仍在MA60上方"** 的方式，将信号提前，同时避免频繁假突破。

### 为何 CS 模型是性能瓶颈？

择时模型已近乎完美地分离牛熊状态（见第二节数据）。在 slots>0 的时段内，进一步提升收益依赖于 CS 模型选出更好的个股。ICIR ~1.0 已处于 A 股因子挖掘的高水平，继续提升空间有限。

### 为何用固定槽位而非等权？

等权重（1/n）在持仓数变化时（如从 6 只变 7 只）会触发所有持仓的级联再平衡，产生大量小额交易。固定槽位（总值/MAX_SLOTS）仅影响新增/退出的那只，大幅减少摩擦成本（实测节省约 3% 的年化摩擦）。
