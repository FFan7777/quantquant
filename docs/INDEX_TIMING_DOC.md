## 1. 模型概述

本模型专为**大盘仓位控制**设计，通过预测沪深300指数的中期走势，输出三档仓位信号（满仓 / 半仓 / 空仓），供上层选股策略动态调整持仓规模。

```
                      ┌────────────────────┐
 技术/宏观/资金特征 ──▶│   XGBoost 分类器   │──▶ pred_prob
                      └────────────────────┘
                                 │
                    ┌────────────▼─────────────┐
                    │    MA 硬覆盖（安全网）     │
                    │ close<MA20 → 强制空仓      │
                    │ MA20-MA60  → 强制半仓      │
                    │ close≥MA60 → ML 自由决定   │
                    └────────────┬─────────────┘
                                 │
                    ┌────────────▼─────────────┐
                    │    仓位档位映射            │
                    │  满仓（20槽）              │
                    │  半仓（10槽）              │
                    │  空仓（0槽）               │
                    └──────────────────────────┘
```

**设计目标**：
- 规避大盘系统性下行风险（熊市缩仓）
- 捕捉趋势性上涨机会（牛市扩仓）
- 可与截面选股模型组合：择时决定槽位总数，选股决定配置哪些股票

## 2. 设计架构

### 数据切分

| 数据集 | 时间范围 | 用途 |
|--------|---------|------|
| 训练集 | 2016-01 ~ 验证集前 rebal_freq 日 | XGBoost 拟合 |
| 验证集 | 默认 2022 全年 | 超参调整（Youden J 阈值扫描） |
| 测试集 | 2023-02 ~ 2025-12 | 样本外评估（含 20 日隔离期） |

> **Purged Split**：训练集末尾向前裁剪 `rebal_freq=15` 个交易日，彻底避免前向标签泄露。

### XGBoost 配置（防过拟合）

| 参数 | 值 | 说明 |
|------|-----|------|
| `n_estimators` | 150 | 固定轮次，不用早停 |
| `max_depth` | 2 | 仅捕获 2 阶交互，防记忆 |
| `min_child_weight` | 30 | 叶节点至少 30 样本才分裂 |
| `reg_lambda` | 15.0 | 强 L2 正则 |
| `reg_alpha` | 1.5 | 强 L1 正则 |
| `colsample_bytree` | 0.60 | 每棵树随机 60% 特征 |

> **为何不用早停**：时序验证集（单一熊市年 2022）方差极大，第 1 棵树即触发早停 → n_trees=0。改为固定轮次 + 强正则化。

## 3. 特征工程

精选 **16 个中长期稳健信号**，剔除短期噪声特征（ret_1d/3d/5d/10d，vol_5d）：

| 类别 | 特征 | 说明 |
|------|------|------|
| 中期动量 | `ret_20d`, `ret_60d`, `ret_120d` | 20/60/120 日累计收益 |
| 波动率状态 | `vol_20d`, `vol_60d` | 年化波动率（20/60日窗口） |
| MA 偏离 | `close_vs_ma20/60/120/250` | 收盘价相对各均线偏离度 |
| 市场宽度 | `breadth_pct_ma20` | 个股中 close>MA20 的比例 |
| 宏观 | `m2_yoy`, `pmi_vs_50`, `shibor_3m` | M2同比 / PMI偏50 / 利率 |
| 北向资金 | `north_20d` | 20 日累计北向净流入 |
| 季节性 | `month_sin`, `month_cos` | 月份正弦/余弦编码 |

> **PMI 处理**：发布月 M 数据→下月第 1 交易日才可用（保守 1 个月发布滞后），严格保持 PIT（Point-in-Time）。

## 4. 标签设计实验

共尝试三种预测目标，结果差异显著：

### 4.1 direction（原始方向标签）—— 接近随机游走

```
label = 1  若  close_{T+15} > close_T
label = 0  否则
```

- 正样本率：**53.3%**（接近 50% 随机游走）
- test_AUC：**0.40-0.50**（≤0.5 = 模型无预测能力或逆预测）
- 根本问题：CSI300 15 日方向无法用技术/宏观特征稳定预测

### 4.2 ma60_state（MA60 状态标签）—— **最优，推荐**

```
label = 1  若  close_{T+15} > MA60_{T+15}
label = 0  否则
```

- 正样本率：~60%（MA60 状态有显著持续性）
- test_AUC：**0.59-0.76**（远高于 direction）
- 为何有效：
  - MA60 状态具有高度自相关性（动量持续性）
  - `close_vs_ma60` 特征与标签之间存在物理稳定关系
  - 不依赖宏观制度假设，跨牛熊均成立

### 4.3 meta_label（Lopez de Prado 元标签）—— 失败

```
# 主模型（MA 规则）：close ≥ MA20 → 发出做多信号
# 元标签（ML）：预测"该 MA 信号是否会盈利"
label = 1  若  primary_active(T) AND fwd_ret > 0
label = 0  若  primary_active(T) AND fwd_ret ≤ 0
label = NaN  若  close < MA20（不训练）
```

- 正样本率：**52.9%**（MA 过滤几乎无效，仍近随机）
- test_AUC：**0.285-0.308**（逆预测）
- 失败原因：
  1. MA 活跃期过滤不能解决"15 日收益方向≈随机游走"的根本问题
  2. 2022 年熊市验证集样本崩溃（仅 79 个 MA 活跃样本），换 2021 年验证集后 val_AUC=0.640 但 test_AUC 仍逆预测
  3. 与 direction 标签面临同样的制度迁移问题（宏观特征方向在测试期反转）

## 5. 仓位决策逻辑

### Step 1：ML 信号映射

**模式 A（默认）：绝对阈值**
```
pred_prob ≥ 0.60  →  满仓（20 槽）
pred_prob ≥ 0.45  →  半仓（10 槽）
pred_prob  < 0.45  →  空仓（0 槽）
```

**模式 B：滚动百分位（`--use_rolling_pct`）**
- 将当期概率与近 30 期历史相对排名比较，消除概率跨期漂移
- 百分位 ≥ 70% → 满仓；≥ 35% → 半仓；< 35% → 空仓
- **注意**：当 test_AUC < 0.5 时，百分位模式会放大逆预测信号，结果更差

### Step 2：MA 硬覆盖（安全网）

| 模式 | 规则 | 适用场景 |
|------|------|---------|
| `hard3`（默认）| close<MA20 → 强制空仓；MA20-MA60 → 强制半仓；close≥MA60 → ML 自由 | 所有场景 |
| `ma20only` | close<MA20 → 强制空仓；MA20+ → ML 自由 | ma60_state 标签 |
| `soft` | ML 极低置信 + MA 深跌才覆盖 | 诊断用 |
| `none` | 关闭覆盖，纯 ML | 诊断用（2022 年亏损 -21.57%） |

> **hard3 覆盖不可去除**：关闭后 WFO 2022 亏损 -21.57%（vs 开启后 -0.42%）

## 6. 训练与评估框架

### Walk-Forward Optimization（WFO）

每次以截至 `year` 为训练+验证，`year+1` 为测试，滚动 4 个窗口：

```
WFO 2022：训练≤2020  验证=2021  测试=2022
WFO 2023：训练≤2021  验证=2022  测试=2023
WFO 2024：训练≤2022  验证=2023  测试=2024
WFO 2025：训练≤2023  验证=2024  测试=2025
```

### 评估指标

| 指标 | 说明 |
|------|------|
| **AUC** | 分类能力（> 0.5 才有意义，> 0.6 较强） |
| **年化收益** | 择时策略年化超额（含交易成本） |
| **Sharpe** | 风险调整后收益 |
| **MaxDD** | 最大回撤 |
| **ML 胜 MA 基准** | ML 年化是否超越纯 MA 三档规则的次数 |

### MA 纯规则基准（`simulate_ma_baseline`）

```
close < MA20   →  空仓
MA20 ≤ close < MA60  →  半仓
close ≥ MA60   →  满仓
```

这是最简单的趋势跟随规则，也是 ML 模型需要超越的基准线。

## 8. 使用方法

### 基础用法

```bash
# 最优配置（ma60_state 标签 + hard3 MA 覆盖）
python index_timing_model.py --label_type ma60_state

# 快速测试（跳过 WFO，仅测试集）
python index_timing_model.py --label_type ma60_state --no_wfo

# 原始方向标签（基准对比）
python index_timing_model.py

# 元标签实验（非熊市验证年）
python index_timing_model.py --label_type meta_label --val_year 2021
```

### 全部参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--label_type` | `direction` | `direction` / `ma60_state` / `meta_label` |
| `--ma_override` | `hard3` | `hard3` / `ma20only` / `soft` / `none` |
| `--val_year` | `2022` | 验证集年份（meta_label 建议用 2021） |
| `--rebal_freq` | `15` | 持有期（交易日） |
| `--threshold_full` | `0.60` | 满仓概率阈值 |
| `--threshold_half` | `0.45` | 半仓概率阈值 |
| `--n_estimators` | `150` | XGBoost 树数量 |
| `--max_depth` | `2` | XGBoost 树深度 |
| `--use_rolling_pct` | `False` | 启用滚动百分位仓位映射 |
| `--no_wfo` | `False` | 跳过 WFO（快速模式） |

## 10. 工程细节与踩坑

### 早停问题
- 时序验证集（单一年份）在熊市年方差极大
- 用 2022 年做验证集时，第 1 棵树就触发早停 → n_trees=0，模型无法训练
- **解决**：关闭早停，改用固定轮次 + 强正则化（reg_lambda=15, reg_alpha=1.5）

### meta_label 仿真 Bug（已修复）
- `test_panel.dropna(subset=['label'])` 会过滤掉 `close<MA20` 的天（meta_label 下这些天 label=NaN）
- 导致调仓时间表只包含 MA 活跃期 → 空仓期被跳过 → 净值曲线严重失真 → MA 基准显示 0% 空仓
- **修复**：分离两个面板：
  - `test_panel_cls`：有标签行，用于 AUC 计算
  - `test_panel_sim`：全部特征可用行，用于仿真（确保空仓期正确进入时间表）

### 滚动百分位的陷阱
- 当 AUC < 0.5（模型逆预测）时，滚动百分位会**放大**逆预测信号
- 模型一致地把"最坏时机"打上高概率 → 百分位排名高 → 满仓 → 亏损更多
- 绝对阈值模式受影响较小（中低概率 → 空仓/半仓 → 损失有限）

### PMI 月度数据滞后
- 月份 M 的 PMI 数据实际在 M+1 月初发布
- 代码中保守处理：当月数据下月第 1 个交易日起才可用
- 不做此处理会产生前视偏差，导致训练 AUC 虚高

## 11. 核心结论

### 为何 ML 很难超越简单 MA 规则？

1. **目标接近随机游走**：CSI300 15 日方向正样本率 53.3%，理论 AUC 上限约 0.55，难以用技术/宏观特征突破
2. **样本量极少**：约 1500 个训练调仓期 vs 600 个模型参数（参数:样本=1:2.5），过拟合几乎不可避免
3. **制度性冲击不可预测**：2022 年（新冠封城 + 美联储加息）、2023-2025 年（政策驱动牛市）等非典型行情无法从技术面预前捕获

### 各方案最终排名

| 排名 | 方案 | 理由 |
|------|------|------|
| 1 | **MA 三档规则（纯规则）** | 简单、稳健、无过拟合风险，WFO 年年稳定 |
| 2 | **ma60_state + hard3** | 2025 年超越 MA 基准，AUC 显著提升 |
| 3 | direction + hard3 | ML 无增益，与 MA 基准持平或更差 |
| 4 | meta_label | AUC 逆预测，结果最差 |

### 实用建议

- **直接使用 MA 三档规则**：调用 `simulate_ma_baseline` 函数，无需 ML
- **若要引入 ML**：使用 `--label_type ma60_state`，2025 年首次显示超越 MA 的能力
- **不推荐滚动百分位**：仅在 AUC > 0.6 的稳定信号下才有意义
- **不推荐 meta_label**：MA 活跃期过滤无法解决"15 日方向≈随机"的根本问题
- **若整合入选股策略**：用 MA 基准的 `slots` 输出（`simulate_ma_baseline`）而非 ML 的，更稳定可靠
